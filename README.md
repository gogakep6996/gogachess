# ♞ ChessClass — шахматы для обучения

Веб-приложение для онлайн-уроков шахмат: интерактивная доска с реальной синхронизацией, встроенный чат, peer-to-peer аудио (WebRTC) и движок **Stockfish** для анализа партий и игры с ИИ.

## ✨ Возможности

| | |
|---|---|
| 🧩 Доска | Интерактивная доска с подсветкой ходов, drag & drop, координатами |
| ✏️ Редактор | Учитель включает «Изменить» — фигуры можно свободно расставлять, ученики видят каждое движение в реальном времени. По кнопке «Продолжить» позиция валидируется и игра идёт по правилам |
| 🎓 Комнаты | Публичные (видны в общем списке) и закрытые (доступ только по ссылке). Копирование ссылки в один клик |
| 👥 Роли | Учитель = создатель комнаты. Только он может включать редактор, мьютить участников и сбрасывать позицию |
| 💬 Чат | Сохраняется в БД, история подгружается при входе |
| 🎙 Аудио | WebRTC mesh — звук идёт напрямую между участниками. **Без сторонних платформ** |
| 🔇 Управление | Учитель может замьютить отдельного ученика или сразу всех |
| 🤖 Движок | Stockfish 16 NNUE прямо в браузере (Web Worker). Уровень сложности 0–20, авто-анализ, игра против ИИ |
| 🌗 Темы | Светлая/тёмная — переключатель в шапке |
| 🔐 Авторизация | Регистрация по email **или** номеру телефона + пароль (JWT-cookie) |

## 🚀 Стек

- **Next.js 15** (App Router) + **React 19**
- **TypeScript** (strict)
- **Tailwind CSS** + кастомная палитра, основанная на классических цветах деревянной доски (`#f0d9b5`/`#b58863`)
- **Socket.IO** на собственном Node-сервере (`server/index.ts`) — синхронизация ходов, чат, **сигналинг WebRTC**
- **Prisma** + **PostgreSQL** (`DATABASE_URL`, см. `.env.example`)
- **chess.js** — валидация ходов и FEN
- **Stockfish.js (NNUE)** — движок через Web Worker

## 📦 Запуск

```bash
# 1) Установить зависимости
npm install

# 2) Скопировать переменные окружения
cp .env.example .env
# (Windows PowerShell)
copy .env.example .env

# 3) Поднимите PostgreSQL локально (или задайте свой DATABASE_URL) и синхронизируйте схему
npm run db:generate
npx prisma migrate deploy

# 4) Запустить dev-сервер (Next.js + Socket.IO в одном процессе)
npm run dev
```

Откройте http://localhost:3000

> **HTTPS для микрофона.** Браузеры дают доступ к микрофону только на `localhost` или по HTTPS. Для тестов через локальную сеть используйте, например, `mkcert` + Caddy/nginx, либо туннель `ngrok http 3000`.

## 🛡 Production

```bash
npm run build
npm start  # NODE_ENV=production tsx server/index.ts
```

### TURN-сервер для WebRTC

Публичный STUN от Google работает у ~85% пользователей. Для надёжной работы за NAT поднимите свой **coturn** и пропишите его в `.env`:

```env
NEXT_PUBLIC_ICE_SERVERS='[
  {"urls":"stun:stun.l.google.com:19302"},
  {"urls":"turn:turn.example.com:3478","username":"user","credential":"pass"}
]'
```

## 🧪 Сценарий «попробовать»

1. Откройте сайт в **двух разных браузерах** (например, Chrome + Firefox в инкогнито).
2. В первом — зарегистрируйтесь как «Учитель», создайте публичную комнату.
3. Скопируйте ссылку из комнаты, откройте во втором — зарегистрируйте «Ученика», войдите в комнату.
4. Учитель → «Изменить позицию» → перетащите фигуры → «Продолжить» — оба окна синхронизированы.
5. Ученик попробует сделать ход — будет валидирован по правилам.
6. Подключите аудио в обоих окнах, проверьте кнопку «Мьют» у учителя.
7. На странице **Анализ** запустите Stockfish — выберите уровень и сыграйте партию.

## 📝 Заметки по дизайну
